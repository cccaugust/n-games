# ローゼンケーニッヒ（Rose King）簡略版

2人専用の陣取り対戦。1台のタブレットで対面プレイ。薔薇戦争モチーフの「王の駒を動かして自分のバラを並べる」ゲーム。

## 概要

9×9盤の中央に王の駒（👑）が置かれ、両プレイヤーが交互に王を動かして、移動先に自分のバラタイル（🌹/🌷）を置きます。最終的に **自分のバラの最大連結数** が多い人の勝ち。

オリジナルは「カードで方向と距離を決める」ゲームですが、本版では1画面対面プレイのため **手札非公開要素を排除**し、自由に方向（8方向）と距離（1〜3マス）を選べる方式に簡略化。

## 操作方法

- 自分の番で、盤面の **黄色く光るマス（移動可能マス）** をタップ
- 王がそこに移動 → 同時に自分のバラを配置
- 相手のバラに移動した場合は **捕獲**（赤縁取りハイライト）

## ゲームルール

### 移動
- 王は **8方向のいずれかに 1〜3マス** 直線移動
- **自分のバラの上には乗れない**（途中も塞がれる）
- 相手のバラの上は乗れる（**捕獲**）が、それ以上は進めない
- 空きマスは通り過ぎ可能

### ターン
- 全30ターン（各プレイヤー15ターン）
- どちらかが動けない場合は片方が連続プレイ
- 両者とも動けない場合は試合終了

### 勝敗
- 終了時に **自分のバラの最大連結成分**（縦横斜めの8方向で隣接）を計算
- 多い人の勝ち
- タイブレーク: 置いたバラの数 → 捕獲数

## 構成

- `index.html`: 自己完結HTML+CSS。SVG盤面、モーダル類
- `game.js`: 状態管理、移動可能マス計算、BFS連結成分検出、レンダリング、効果音
- `README.md`: このファイル

## 状態管理

- `state.board`: 9x9のplayerId or null
- `state.king`: { c, r } 王の位置
- `state.players`: { id, name, color, accent, icon }
- `state.captures`: { 1, 2 } 捕獲数
- `state.reachable`: 現在のプレイヤーが移動可能なマスのリスト
- `state.turnsLeft`: 残ターン数
- `state.phase`: `'menu'` / `'playing'` / `'over'`
- `state.winner`: 勝者プレイヤーIDの配列
- `state.scores`: 各プレイヤーの { cluster, stones, captures }

## 移動可能マスの計算

8方向 × 距離1〜3でループ:
- 自分のタイル → そこで打ち切り（着地不可）
- 空きマス → 着地可能、引き続き探索
- 相手タイル → 着地可能（捕獲）、その先は探索しない

## 連結成分検出

各プレイヤーのタイルにBFSをかけて、最大のグループを返します。8方向（縦横斜め）隣接で同色なら連結とみなします。

## レスポンシブ対応

- 縦置きタブレット: P2パネル(反転) / 盤面 / HUD / P1パネル
- 横置きでもOK
- スマホでも遊べる(SVGがレスポンシブ)

## 効果音

WebAudio API合成。

- `move`: 王の移動音（短いスライド）
- `place`: バラ配置音（2音重ね）
- `capture`: 捕獲時のキラキラ音
- `invalid`: 不正タップ
- `tap` / `win`
