# コリドール（Quoridor）

2人専用の戦略アブストラクト。1台のタブレットで対面プレイ。9×9のマスを進んで反対側のゴールへ最初に到達した人が勝ちですが、壁で相手の道をふさぐのが醍醐味です。

## 概要

- 自分のコマを反対側のゴール行に運ぶレース
- 各プレイヤー10枚の壁を持っていて、毎ターン「コマを動かす」か「壁を置く」を選択
- 壁は2マス分の長さで、コマの通り道をふさぐ
- ただし、相手のゴールへの道を完全にふさぐ壁は置けない（必ず通り道を残す）
- 相手のコマと隣り合ったらジャンプ可能、ジャンプの先に壁があれば斜めに回り込める

## 操作方法

### モード切替（HUDの「🧱 壁モード」ボタン）
- **🚶 移動モード（既定）**: 自分のコマをタップ → 移動可能マス（黄色光）が表示 → 移動先タップ
- **🧱 壁モード**: 盤面に薄い壁スロットが現れる → タップで配置（不正な置き方ならエラー表示＆音）

### キーボード
- `Esc`: コマ選択を解除

## ゲームルール

### 移動
- 1ターンに1マス、上下左右のいずれか1方向
- 壁で塞がれている隙間は通れない
- 相手のコマの隣で、その向こうが空いていれば「ジャンプ」（2マス前進）可
- ジャンプ先が壁/盤外で塞がれている場合、相手の左右どちらかに「斜め移動」可

### 壁
- 各プレイヤー10枚
- 縦置き or 横置き、2マス分の長さ
- 既存の壁と重ならない / 交差しない
- **両プレイヤーとも自分のゴールに到達できる経路が残る** ように置く

### 勝利
- 自分のコマがゴール行（P1=row 0、P2=row 8）に到達したら勝ち

## 構成

- `index.html`: 自己完結HTML+CSS。SVG盤面の入れ物、モーダル類
- `game.js`: 盤面状態、移動/壁ロジック、BFS経路チェック、レンダリング、効果音
- `README.md`: このファイル

## 状態管理

- `state.players`: { id, name, color, walls(残数), goal, ... }
- `state.pawns`: [{ player, c, r }]
- `state.walls`: [{ orient: 'h'|'v', c, r }] アンカーは内側交差点
- `state.phase`: `'menu'` / `'playing'` / `'over'`
- `state.currentPlayer`: 1 or 2
- `state.mode`: `'move'` or `'wall'`
- `state.selected`: 自分のコマを選択中か
- `state.movables`: 移動候補リスト

## 壁の座標系

`(orient, c, r)` で表現。c, r は 0〜7（盤の内側交差点）。

- **horizontal wall (c, r)**: cells (c,r)↔(c,r+1) と (c+1,r)↔(c+1,r+1) の縦方向移動を遮る
- **vertical wall (c, r)**: cells (c,r)↔(c+1,r) と (c,r+1)↔(c+1,r+1) の横方向移動を遮る

衝突条件:
- 同じ anchor (c, r) に別の壁: 不可（交差）
- 同じ orient で1マスずれ: 不可（重複）

## 経路チェック

壁を仮置きしてBFS。両プレイヤーがそれぞれのゴール行に到達できなければ非合法。

## レスポンシブ対応

- 縦置きタブレット: 上(P2 反転) / 盤 / HUD / 下(P1)
- スマホでもプレイ可（盤面が画面幅いっぱいに縮小）

## 効果音

WebAudio API合成（外部ファイル不要）。

- `select` / `move` / `jump` / `wall`(2音重ね) / `invalid` / `tap` / `win`(上昇トライアド)
