# パッチワーク（Patchwork）

2人専用の戦略タイル配置ゲーム。1台のタブレットで対面プレイができます。簡略化MVP版。

## 概要

20種のパッチ（ポリオミノ）を市場から買って、9×9のキルトを埋めるゲーム。買い物にはボタン（お金）と時間がかかります。時間トラックで後ろにいる人が常に手番なので、「安いけど時間がかかるパッチ」と「高いけど時間が短いパッチ」をどう選ぶかが鍵。

## 操作方法

### ターンの選択肢
1. **パッチを買う**: マーケットの3枚から1つタップ → 配置パネルで「↻回転 / ⇋反転」 → キルトの空きマスをタップで配置
2. **進む**: HUDの「▶ 進む」ボタンで相手の1マス先まで移動。進んだ分だけボタンをもらえる

### 配置時のキーボードショートカット
- `R`: 回転
- `F`: 反転
- `Esc`: キャンセル

## ゲームルール

### ターン
- **時間トラックで後ろにいる人** が常に手番
- 同マスに重なってる時は、後から来た人が先に動く（直近に動いた人と逆）

### パッチを買う
- 中央トークンから時計回りに3枚だけが購入対象
- 🪙コストを払う（足りないと買えない）
- 9×9キルトに置く（重なり/盤外NG、回転・反転OK）
- 買ったパッチの位置に中央トークンが移動 → 次の3枚が変わる
- ⏰時間コスト分だけ自分の時間トラックを進める
- 通過した🪙マスごとに、収入分のボタンをゲット

### 進む（パス）
- 相手の1マス先まで自分のマーカーを進める
- 進んだマス数だけ🪙ボタンを獲得（時間トラックの🪙マス通過ボーナスも別途）

### 終了
- 両者が🏁(時間=25)に到達したら終了

### 得点
- **スコア = 持ってる🪙 − 空きマス数 × 2**
- スコアが多い人の勝ち（同点は引き分け）

## 簡略化ポイント

オリジナルのPatchworkから以下を簡略化：
- 5枚の特殊「1×1パッチ」（leather patch）は省略
- 「7×7ボーナス」（最初に7×7を埋めた人 +7点）は省略
- 時間トラック33→25、🪙マス9→5
- パッチ33→20

戦略の核（時間vsお金、買うか進むか）は維持。

## 構成

- `index.html`: 自己完結HTML+CSS。デュアルキルト、マーケット、時間トラック、モーダル類
- `game.js`: パッチ定義、状態管理、回転/反転、配置検証、レンダリング、効果音
- `README.md`: このファイル

## 状態管理

- `state.players`: { id, name, color, buttons, time }
- `state.quilts`: { 1: 9x9 grid, 2: 9x9 grid } 各セルは `{patchId, color, hasButton}` or null
- `state.incomes`: { 1, 2 } 各プレイヤーのボタン収入合計
- `state.patches`: 残ってるパッチの循環配列
- `state.marketIndex`: 中央トークン位置（買える3枚の先頭）
- `state.selectedPatchId` / `state.selectedShape`: 配置中のパッチ
- `state.phase`: `'menu'` / `'playing'` / `'over'`
- `state.lastMover`: 最後に動いたプレイヤー（タイブレーク用）

## レスポンシブ対応

- 縦置きタブレット: P2パネル(反転) / P2キルト / マーケット / 時間トラック / P1キルト / P1パネル
- スマホでもプレイ可（セル22px〜26pxに自動縮小）
- P2キルト本体は反転しない（タップ座標のずれを防ぐため）。パネル文字のみ反転

## 効果音

WebAudio API合成。

- `select` / `rotate` / `place` / `coin`(ボタン獲得時、複数回チャイム) / `skip` / `invalid` / `tap` / `win`
